# 可控成本自动化内容运营目标

本项目的最终目标不是单纯生成小红书笔记，也不是单纯生成短视频，而是建设一个可控成本、质量达标、可审核、可复盘的自动化内容运营系统。

目标出口包括：

- 小红书图文 / 视频
- 公众号文章
- 视频号短视频
- 抖音短视频
- 后续可扩展到知乎、B 站、私域社群等

目标收益模式包括：

- 平台流量分成
- 商品带货佣金
- 小程序、课程、短剧、工具等 CPS / CPA 分佣
- 账号增长后的商单
- 公众号或私域沉淀后的服务转化

所有收益预期都必须视为假设，需要用数据验证，不能写进系统逻辑里当确定事实。

## 产品北极星

用可控的人力和 API 成本，持续产出“足够好看、可发布、可复盘、可优化”的内容资产，并逐步找到稳定 ROI 为正的内容赛道和平台组合。

低成本不等于无成本。免费工具和本地生成只适合验证流程；真正面向发布时，素材、配音、图片、视频生成可以付费，只要质量提升能够带来更高通过率、播放、转化或复用价值。

## 系统边界

系统负责：

- 抓取热点和候选选题。
- 生成平台无关的内容 brief。
- 为不同平台生成适配稿。
- 生成封面、配图、视频文稿、字幕、视频。
- 生成发布包。
- 记录成本、来源、发布时间、平台、数据表现。
- 根据数据反馈优化选题和模板。

系统暂不默认负责：

- 绕过平台规则自动发布。
- 未授权搬运素材。
- 夸张收益承诺。
- 无审核批量发布敏感内容。
- 自动挂不相关商品。

## 平台定位

### 小红书

- 强项：图文种草、生活方式、消费决策、女性用户、搜索流量。
- 内容形态：封面强、标题强、信息密度高、可收藏。
- 系统重点：封面、标题、结构化图文、商品匹配、搜索关键词。

### 公众号

- 强项：长文、沉淀、私域、服务转化。
- 内容形态：深度文章、专题合集、资料包、观点复盘。
- 系统重点：长文结构、事实来源、摘要、封面、排版、系列化栏目。

### 视频号

- 强项：微信生态、熟人/半熟人传播、公众号联动。
- 内容形态：短视频 + 私域承接。
- 系统重点：短口播、字幕、封面、公众号文章互链。

### 抖音

- 强项：大流量、强推荐、短视频转化。
- 内容形态：强钩子、快节奏、15-60 秒。
- 系统重点：前 3 秒、字幕节奏、画面质量、商品相关性、数据复盘。

## 内容资产模型

一次热点生成不应该只产出一篇“小红书文案”，而应产出一组平台无关资产：

- `hotspot`：热点标题、来源、时间、热度、URL。
- `content_brief`：事实点、角度、风险点、受众、可视化方向。
- `article_plan`：长文结构，供公众号等使用。
- `social_post_plan`：短图文结构，供小红书等使用。
- `video_plan`：口播稿、字幕、分镜、图片关键词、时长。
- `commerce_plan`：可选商品方向、佣金假设、禁挂规则。
- `platform_packages`：不同平台的标题、正文、封面、视频、标签、发布说明。
- `performance`：发布后的播放、阅读、点击、转化、收入、成本。

## 质量 / 成本 / ROI 原则

- 先定义最低可发布质量，再在这个质量线之上优化成本。
- 先用规则和模板解决稳定结构问题，再把 LLM 用在高价值步骤。
- 默认不在页面加载、预览、刷新列表时调用付费 API。
- 生成前明确成本，生成后写回成本。
- 视频素材搜索和 TTS 设置调用上限，但允许配置更高质量的付费 provider。
- 优先复用缓存和已生成资产。
- 数据表现差的选题、素材源和模板要淘汰，不继续烧 API。
- 免费源、本地源、低价源、高质量付费源都应该以 provider 方式接入，方便 A/B 对比质量、成本和效果。

## 素材和供应商策略

- 腾讯 WIMGS 属于低价图片搜索源，可以作为过渡和备选，但不能默认代表最终质量。
- 如果图片质量差，应优先优化 provider 选择、评分过滤、素材授权和行业图库，而不是用大黑块遮丑。
- TTS 同理：macOS `say` 只是流程验证工具，不是发布质量目标。
- 后续应支持更高质量的 TTS、图片、视频 provider，并在每条内容里记录 provider、单次成本、质量评分和转化表现。

## 质量和风控原则

- 不能直接搬运平台或他人素材。
- 热点内容必须保留来源和风险提示。
- 财经、房产、健康、公共事件不做确定性建议。
- 商品链接必须和内容强相关。
- AI 生成内容必须可审核、可编辑、可追溯。
- 自动发布只能在合规接口或用户明确授权下进行。

## MVP 路线

### 阶段 1：内容资产稳定

- 热点抓取稳定。
- 草稿 JSON 平台中立。
- 小红书图文和视频发布包可用。
- Web 工作台可审核、可编辑、可生成。

### 阶段 2：视频质量达标

- 专业 TTS provider。
- 更好的分镜模板。
- 更好的图片源和素材评分。
- 字幕和口播稳定同步。

### 阶段 3：多平台发布包

- 小红书图文/视频包。
- 公众号文章包。
- 抖音/视频号视频包。
- 每个平台独立标题、标签、封面和说明。

### 阶段 4：数据闭环

- 手动录入或半自动抓取平台数据。
- 记录每条内容成本、播放、阅读、点击、转化、收入。
- 计算 ROI、筛选赛道、淘汰低效模板。

### 阶段 5：半自动运营

- 批量生成候选。
- 人工审核通过后定时发布。
- 根据数据反馈调整选题池、脚本模板、平台策略。

## 当前优先级

短期不要追求全自动发布，先把下面三件事做扎实：

1. Web 工作台重构：清晰审核、编辑、生成、发布包流程。
2. 视频质量提升：文稿、字幕、TTS、配图先达到可发布水平。
3. 成本和效果记录：每条内容知道花了多少钱、发到哪里、效果如何。
